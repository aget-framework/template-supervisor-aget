## Contribution Flow

### 1. Experiment Here
- Try radical ideas in `workspace/`
- Test breaking changes safely
- Learn from failures

### 2. Validate
- Prove value through usage
- Document patterns discovered
- Extract reusable components

### 3. Graduate
- Move proven patterns to template
- Ensure backward compatibility
- Add comprehensive tests

### 4. Release
- Patterns become public in template
- Community can fork and use
- Maintain stability
