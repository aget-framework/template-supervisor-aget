# AGET Technical Glossary

## Core Concepts

### Cognitive Spectrum
**Technical Term**: Cognitive Modality Taxonomy
**Definition**: A hierarchical classification system defining five distinct operational contexts where AGET instances provide cognitive augmentation, ranging from data-only analysis to meta-orchestration of multiple agent systems.

### Evolution Tracking
**Technical Term**: Temporal Decision Provenance System
**Definition**: A versioned, timestamped ledger recording architectural decisions (DEC), empirical discoveries (DISC), and component extractions (EXT) to maintain institutional knowledge and decision rationale.

### Pattern Graduation
**Technical Term**: Progressive Pattern Maturation Pipeline
**Definition**: The formal process by which experimental patterns in private repositories undergo validation, refinement, and eventual promotion to public framework status with backward compatibility guarantees.

### Strategic Governance
**Technical Term**: Hierarchical Authority Delegation Model
**Definition**: The architectural pattern where aget-aget maintains authoritative control over vision and strategic direction while aget-cli-agent-template provides stable implementation interfaces.

### Bridge Extraction
**Technical Term**: Component Isolation and Repackaging Protocol
**Definition**: The systematic process of extracting workspace experiments into standalone products with proper dependency management, interface contracts, and documentation.

### Session Management
**Technical Term**: Contextual State Persistence Framework
**Definition**: The protocol suite managing agent lifecycle states (wake/work/wind-down/sign-off) with automatic context preservation, checkpoint creation, and session continuity.

### Conversational Infrastructure
**Technical Term**: Natural Language Command Interface Layer
**Definition**: The abstraction layer translating conversational directives ("hey", "wake up", "wind down") into systematic agent behaviors and state transitions.

### Modality Shifting
**Technical Term**: Dynamic Context Adaptation Mechanism
**Definition**: The capability of an AGET instance to transition between cognitive modalities (standalone → owner → contributor → analyst → meta) based on operational requirements.

### Pattern Repository
**Technical Term**: Reusable Behavior Component Library
**Definition**: A structured collection of validated, parameterized behavioral templates that can be instantiated across different AGET deployments with configuration-based customization.

### Specification-Driven Development
**Technical Term**: Contract-First Implementation Paradigm
**Definition**: The methodology where functional specifications precede implementation, with specifications serving as the authoritative contract between human intent and agent execution.

### Meta-Governance
**Technical Term**: Recursive Orchestration Architecture
**Definition**: The highest cognitive modality where an AGET instance manages and coordinates multiple subordinate AGET instances, each potentially operating in different modalities.

## Architectural Patterns

### Fork-First Design
**Technical Term**: Divergent Customization Architecture
**Definition**: The intentional design pattern optimizing for repository forking and customization rather than monolithic configuration, enabling organizational sovereignty over agent behavior.

### Three-Tier Degradation
**Technical Term**: Graceful Capability Regression Protocol
**Definition**: The architectural pattern ensuring functionality across three operational tiers: Rich (full tooling) → Standard (basic tooling) → Basic (guaranteed minimal functionality).

## Cognitive Unix Concepts

### Semantic Pipes
**Technical Term**: Meaning-Preserving Transform Connectors
**Definition**: The evolution of Unix pipes (`|`) from text stream processors to semantic stream processors, where each pipeline stage transforms meaning rather than just characters, enabling compositions like `understand | reason | create`.

### Fractal Composability
**Technical Term**: Multi-Scale Tool Composition Architecture
**Definition**: The property where "tools" can be Unix commands, LLM calls, hybrid pipelines, or entire agents, all composable through the same semantic pipe interface, creating nested hierarchies of cognitive processing.

### Semantic Streams
**Technical Term**: Structured Meaning Flow Protocol
**Definition**: Data streams that carry semantic content with preserved meaning across transformations, using formats like JSON-LD, semantic markdown, or other meaning-preserving serializations.

### Cognitive Pipeline
**Technical Term**: Interpretive Processing Chain
**Definition**: A sequence of semantic transformations where each stage can be deterministic (Unix tools) or probabilistic (LLMs), connected by semantic pipes that preserve and transform meaning: `find_bug | explain_why | generate_fix | verify_safe | apply_patch`.

### Universal Semantic Interface
**Technical Term**: Meaning as Lingua Franca
**Definition**: The philosophical evolution from "text is the universal interface" to "meaning is the universal interface", where all cognitive tools speak through common semantic protocols.

### Predictable Unpredictability
**Technical Term**: Bounded Stochastic Behavior Pattern
**Definition**: The architectural pattern of containing LLM unpredictability within small, testable units with declared confidence bounds, enabling reliable composition of probabilistic components.

### Checkpoint States
**Technical Term**: Contextual Configuration Persistence
**Definition**: Saved states of LLM context and configuration that can be loaded to reproduce specific semantic behaviors, addressing the challenge of probabilistic variability in cognitive systems.

### Semantic Grep
**Technical Term**: Meaning-Based Search Operator
**Definition**: Search operations that match by semantic meaning rather than textual patterns, enabling queries like "find all code expressing error handling" regardless of syntax.

### Cognitive Cron
**Technical Term**: Scheduled Understanding Tasks
**Definition**: The extension of Unix cron to schedule semantic processing tasks, enabling periodic cognitive operations like "analyze codebase for patterns every night" or "summarize daily changes".

### Pipe Dream Amplified
**Technical Term**: Enhanced Composability Vision
**Definition**: The philosophical extension of Unix's composable tools dream into the cognitive era, where deterministic and interpretive processing combine through semantic pipes to create programmable thinking partners.
