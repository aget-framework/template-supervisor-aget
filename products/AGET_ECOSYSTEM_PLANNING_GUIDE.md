# AGET Ecosystem Planning Guide
*A framework for identifying and prioritizing future X-aget projects*

## Purpose
This guide helps AGET users systematically identify which specialized agents would most benefit their workflow and goals. Rather than creating agents randomly, this framework ensures each new X-aget serves a specific purpose in your cognitive ecosystem.

## Planning Process

### Step 1: Understand Your Current State
Before creating new agents, map what you have:
- List existing X-aget projects
- Identify their primary functions
- Note integration points between them
- Recognize gaps in your workflow

### Step 2: Define Your Vision
What are you optimizing for?
- **Productivity**: Automating repetitive tasks
- **Creativity**: Enhancing ideation and creation
- **Knowledge**: Building and sharing understanding
- **Quality**: Ensuring excellence in output
- **Growth**: Personal/professional development

### Step 3: Identify Patterns
Look for recurring needs:
- Tasks you do >3 times per week
- Workflows with multiple manual steps
- Information you frequently synthesize
- Decisions you repeatedly make
- Knowledge you often reference

## Agent Categories Framework

### 🎯 Strategic Cognitive Partners
Long-term thinking and planning agents that shape your cognitive landscape.

**Examples:**
- **memory-aget**: Cross-session context retention
- **synthesis-aget**: Pattern recognition across projects
- **coach-aget**: Personal development partner
- **strategy-aget**: Long-term planning assistant

**When to build:** When you need consistent cognitive support for complex, ongoing challenges.

### 🛠️ Productivity Amplifiers
Automation and efficiency agents that handle repetitive work.

**Examples:**
- **workflow-aget**: Multi-step process automation
- **review-aget**: Quality assurance automation
- **docs-aget**: Documentation generation
- **deploy-aget**: Deployment automation

**When to build:** When manual processes consume >2 hours/week or involve >5 repetitive steps.

### 🌐 Ecosystem Enhancers
Agents that connect and cultivate your broader knowledge system.

**Examples:**
- **garden-aget**: Knowledge cultivation
- **teacher-aget**: Pattern extraction and sharing
- **research-aget**: Deep investigation
- **curator-aget**: Content organization

**When to build:** When information is scattered or knowledge transfer is inefficient.

### 🎨 Creative Collaborators
Agents that enhance creative and innovative work.

**Examples:**
- **storyteller-aget**: Narrative creation
- **composer-aget**: System orchestration
- **designer-aget**: Visual/structural design
- **experimenter-aget**: Hypothesis testing

**When to build:** When you need consistent creative partnership or innovation support.

### 🔍 Specialized Domains
Domain-specific agents for particular fields or interests.

**Examples:**
- **music-aget**: Music analysis and creation
- **finance-aget**: Financial tracking and analysis
- **health-aget**: Wellness monitoring
- **learning-aget**: Skill acquisition

**When to build:** When domain expertise would significantly enhance your work.

## Prioritization Matrix

| Impact ↓ / Effort → | Low Effort | Medium Effort | High Effort |
|---------------------|------------|---------------|-------------|
| **High Impact**     | 🟢 Do First | 🟢 Do Second | 🟡 Plan Carefully |
| **Medium Impact**   | 🟢 Quick Wins | 🟡 Consider | 🔴 Probably Skip |
| **Low Impact**      | 🟡 If Easy | 🔴 Skip | 🔴 Definitely Skip |

### Scoring Criteria

**Impact Assessment:**
- Hours saved per week (High: >5, Medium: 2-5, Low: <2)
- Quality improvement (High: Critical, Medium: Notable, Low: Marginal)
- Cognitive load reduction (High: Major, Medium: Moderate, Low: Minor)
- Beauty/joy created (High: Significant, Medium: Some, Low: Minimal)

**Effort Assessment:**
- Setup complexity (Low: <1 day, Medium: 1-3 days, High: >3 days)
- Maintenance needs (Low: Quarterly, Medium: Monthly, High: Weekly)
- Integration requirements (Low: Standalone, Medium: 2-3 touchpoints, High: Many dependencies)

## Implementation Strategy

### Phase 1: Foundation (Months 1-2)
Start with one agent from each category:
1. **Memory agent** (Strategic) - Never lose context
2. **Workflow agent** (Productivity) - Automate biggest time sink
3. **Garden agent** (Ecosystem) - Organize existing knowledge

### Phase 2: Enhancement (Months 3-4)
Add specialized support:
1. **Review agent** - Ensure quality
2. **Synthesis agent** - Connect insights
3. **Domain-specific agent** - Address primary work area

### Phase 3: Orchestration (Months 5-6)
Enable agent collaboration:
1. **Composer agent** - Coordinate other agents
2. **Teacher agent** - Share patterns
3. **Evolution tracking** - System-wide learning

## Anti-Patterns to Avoid

### ❌ The Kitchen Sink
Creating an agent that tries to do everything. Keep agents focused.

### ❌ The Orphan
Creating agents without clear integration points. Ensure connectivity.

### ❌ The Duplicate
Building what already exists. Check aget-template patterns first.

### ❌ The Premature Optimizer
Automating before understanding the process. Manual first, then automate.

### ❌ The Rigid System
Over-constraining agent behavior. Allow for emergence and growth.

## Success Indicators

Your AGET ecosystem is healthy when:
- ✅ Each agent has a clear, distinct purpose
- ✅ Agents complement rather than compete
- ✅ Integration feels natural, not forced
- ✅ You spend more time creating than configuring
- ✅ The system grows more valuable over time
- ✅ Others can understand and adopt your patterns
